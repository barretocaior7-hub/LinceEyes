# 🛡️ HealthTech - Sistema Biométrico de Controle de Acesso Web

Aplicação "Visão de Lince" — login biométrico por reconhecimento facial usando `face-api.js` no navegador e backend em Node.js integrando com Supabase/Postgres.

---

## 📌 Visão Geral

- Objetivo: permitir cadastro e autenticação por face (webcam) sem senhas.
- Principais arquivos: [server.js](server.js), [lib/supabase.js](lib/supabase.js), [public/cadastro.html](public/cadastro.html), [public/login.html](public/login.html), [public/painel-secreto.html](public/painel-secreto.html), [public/js/cadastro.js](public/js/cadastro.js), [public/js/login.js](public/js/login.js).

## 🏛 Arquitetura

- Frontend: páginas estáticas servidas por Express, usam `face-api.js` (via @vladmandic) para extrair descritores faciais no cliente.
- Backend: Node + Express, endpoints REST que gravam/ler do Supabase (Postgres).
- Banco: Supabase/Postgres com tabela `users` (coluna `descriptor` como JSONB).

## 🔌 Endpoints (API)

- `POST /api/auth/login-rh`
   - Descrição: autentica operador RH (demo com usuário fixo `admin_rh`).
   - Body: `{ "username": string, "password": string }`
   - Retorno: `{ token, role }` ou erro 401.

- `POST /api/register` (protegido - RH)
   - Descrição: cadastra colaborador com descritor facial. Requer `Authorization: Bearer <token>`.
   - Body: `{ name, cargo, setor, descriptor }` (descriptor = array de floats)
   - Processo: insere no Supabase e atualiza `employee_code`.
   - Retorno: 201 com `{ message, employee: { id, employeeCode, name, cargo, setor } }`.

- `GET /api/users`
   - Descrição: retorna lista de usuários com `descriptor` desserializado.

Veja a implementação em [server.js](server.js).

## 🧭 Fluxos do Frontend

- Cadastro (RH):
   1. Operador RH faz login em `/cadastro`.
   2. Carrega modelos face-api e ativa câmera.
   3. Captura descritor via `faceapi.detectSingleFace(...).withFaceDescriptor()`.
   4. POST `/api/register` com token RH.

- Login biométrico (usuário):
   1. Página `/login` carrega modelos e busca `/api/users`.
   2. Constrói `faceapi.LabeledFaceDescriptors` e `FaceMatcher`.
   3. Detecta faces em loop e compara; se match, grava flag em `localStorage` e redireciona para `/painel-secreto`.

Funções relevantes (ver arquivos):
- [public/js/cadastro.js](public/js/cadastro.js): `initFaceApi()`, `startCamera()`, handler do form de cadastro.
- [public/js/login.js](public/js/login.js): `init()`, `startVideo()`, loop de detecção e autenticação.

## 🗄️ Modelo de Dados (SQL)

Execute no SQL Editor do Supabase para criar tabela `users`:

```sql
create table if not exists public.users (
   id bigint generated by default as identity primary key,
   employee_code text unique,
   name text not null,
   cargo text not null,
   setor text not null,
   descriptor jsonb not null,
   created_at timestamptz default now()
);
create index if not exists idx_users_employee_code on public.users (employee_code);
```

Inserção de exemplo e atualização do código de colaborador:

```sql
insert into public.users (employee_code, name, cargo, setor, descriptor)
values ('EMP-0001', 'Fulano Silva', 'Analista', 'TI', '[0.1,0.2,0.3]'::jsonb);

update public.users set employee_code = concat('EMP-', lpad(cast(id as text),4,'0')) where employee_code is null;
```

## 🔐 Segurança, Spoofing e Liveness

- Câmera embaçada: aumenta false-negatives — adicionar avisos de UI e pedir recaptura.
- Foto impressa (spoofing): `face-api.js` puro não detecta liveness — recomendações:
   - implementar checks de movimento/pose (pedir virar a cabeça, piscar);
   - usar modelos/serviços de liveness; combinar com MFA para operações sensíveis.
- Armazenamento: preferir salvar descritores (vetores) ao invés de imagens; criptografar colunas sensíveis e limitar acesso via RLS.

## ☁️ Deploy (Render) — Checklist

- Push do código para GitHub.
- Crie Web Service no Render apontando para o repo/branch.
- Runtime: Node (v18+ / 24 está OK se compatível).
- Build Command: `npm install`
- Start Command: `npm start` ou `node server.js`
- Environment Variables (Render → Environment):
   - `SUPABASE_URL` = https://<seu-project-ref>.supabase.co
   - `SUPABASE_SECRET_KEY` = sb_secret_... (Service Role Key — **somente backend**)
   - `SUPABASE_PUBLISHABLE_KEY` = sb_publishable_... (anon/cliente)
   - `SUPABASE_JWKS_URL` = https://<project>.supabase.co/auth/v1/.well-known/jwks.json (opcional)
   - `DATABASE_URL` = opcional (se usar conexão direta ao Postgres)

Nota: removemos `sqlite3` das dependências para evitar erro de binário nativo no deploy (invalid ELF header).

## 🧪 Testes / Exemplos (curl)

- Obter token RH:
```bash
curl -X POST https://<host>/api/auth/login-rh \
   -H "Content-Type: application/json" \
   -d '{"username":"admin_rh","password":"RH@Health2026!"}'
```
- Cadastrar usuário (token RH):
```bash
curl -X POST https://<host>/api/register \
   -H "Content-Type: application/json" \
   -H "Authorization: Bearer <TOKEN>" \
   -d '{"name":"Fulano Silva","cargo":"Analista","setor":"TI","descriptor":[0.1,0.2,0.3]}'
```
- Listar usuários:
```bash
curl https://<host>/api/users
```
